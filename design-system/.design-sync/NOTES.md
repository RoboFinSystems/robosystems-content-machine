# design-system — bootstrap notes

This directory is the **git-versioned source of truth** for the **"RoboSystems Content
Design System"** Claude Design project (`746ae7a4-52ee-439f-a87a-523dbee2f73c`). The project
was originally authored *in* Claude Design (no prior local repo); it was pulled down here on
**2026-06-30** so it can be curated alongside the content pipeline it serves.

It is the brand source of truth for **Stage 2 (Claude Design: deck + thumbnail)** — see the
repo's `template/DESIGN_INSTRUCTIONS.md`, which points the design task at this project and its
`video-deck` / `thumbnail` templates.

## What's here (the curated source)
- `styles.css`, `fonts.css`, `tokens/*.css` — the design language (link `styles.css`).
- `components/{brand,controls,data}/*.jsx` + `*.d.ts` were **not** pulled (see below); `*.prompt.md` not pulled; `*.card.html` preview cards present.
- `templates/{video-deck,thumbnail,blog-post,social-card}/*.dc.html` — the four content templates.
- `fonts/` — Orbitron + Space Grotesk TTFs (reused from the repo's `assets/fonts/` + the app-DS archive; JetBrains Mono loads from Google Fonts via `fonts.css`).
- `assets/` — brand mark (currentColor + white), logo, lockups, partner/source logos, grid texture.
- `readme.md` (full brand guide), `SKILL.md`, `_adherence.oxlintrc.json` (token + component-prop contract).

## Generated / runtime files (not committed — regenerated or re-pulled)
- `_ds_bundle.js`, `_ds_bundle.css` — **regenerated locally** by `just design-build` (`scripts/build.mjs`); gitignored. The `*.card.html` previews reference `../../_ds_bundle.js`, so run the build before opening them standalone.
- `_ds_manifest.json` — the Design-System-pane card index. **Not** regenerated by `scripts/build.mjs` (re-pull it from the project if you add/remove components or templates). Gitignored.
- `templates/*/ds-base.js`, `templates/*/support.js` — Claude Design `<x-dc>` runtime shims.
- per-component `*.d.ts` / `*.prompt.md` — TypeScript types + design-agent usage docs.
- `templates/*/.thumbnail` — preview images.
- `_adherence.oxlintrc.json` here omits the verbose generated `tokenKinds` map (kept the token list + component prop rules).

## Round-trip
- **Pull (refresh from Claude Design):** `DesignSync get_file` per path (no bulk export).
- **Push (publish edits):** `DesignSync finalize_plan` + `write_files` against the projectId,
  then re-arm the `_ds_needs_recompile` sentinel so the project refreshes. Do a *targeted*
  push (only changed files, `deletes: []`) — a full reconciling sync would delete the runtime
  files this repo doesn't mirror (`ds-base.js`/`support.js`, `_ds_manifest.json`, etc.).
  Tokens / CSS / `.dc.html` templates / assets are plain source and take effect on push.
- **Component edits round-trip too:** after editing `components/*.jsx`, run **`just design-build`**
  (Node + `@babel/preset-react`, `scripts/build.mjs`) to regenerate `_ds_bundle.js` — a faithful
  reproduction of the Claude Design `@ds-bundle` format (Babel classic-runtime IIFE, per-component
  `try/catch`, sibling refs → `__ds_scope.X`, sha256-12 `sourceHashes`). **Verified:** rebuilding
  the untouched components reproduces 8/9 of the original hashes byte-for-byte. Then push the fresh
  `_ds_bundle.js`. First `genericize examples` round-trip (templates + bundle) was pushed 2026-06-30.
