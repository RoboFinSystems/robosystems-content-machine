# design-system ‚Äî bootstrap notes

This directory is the **git-versioned source of truth** for the **"RoboSystems Content
Design System"** Claude Design project (`746ae7a4-52ee-439f-a87a-523dbee2f73c`). The project
was originally authored *in* Claude Design (no prior local repo); it was pulled down here on
**2026-06-30** so it can be curated alongside the content pipeline it serves.

It is the brand source of truth for **Stage 2 (Claude Design: deck + thumbnail)** ‚Äî see the
repo's `template/DESIGN_INSTRUCTIONS.md`, which points the design task at this project and its
`video-deck` / `thumbnail` templates.

## What's here (the curated source)
- `styles.css`, `fonts.css`, `tokens/*.css` ‚Äî the design language (link `styles.css`).
- `components/{brand,controls,data}/*.jsx` + `*.d.ts` were **not** pulled (see below); `*.prompt.md` not pulled; `*.card.html` preview cards present.
- `templates/{video-deck,thumbnail,blog-post,social-card}/*.dc.html` ‚Äî the four content templates.
- `fonts/` ‚Äî Orbitron + Space Grotesk TTFs (reused from the repo's `assets/fonts/` + the app-DS archive; JetBrains Mono loads from Google Fonts via `fonts.css`).
- `assets/` ‚Äî brand mark (currentColor + white), logo, lockups, partner/source logos, grid texture.
- `readme.md` (full brand guide), `SKILL.md`, `_adherence.oxlintrc.json` (token + component-prop contract).

## Deliberately NOT mirrored (generated / runtime ‚Äî re-pull or rebuild if needed)
These live in the Claude Design project and are regenerated by a build; they were skipped to
keep the repo clean and the pull cheap:
- `_ds_bundle.js`, `_ds_bundle.css`, `_ds_manifest.json` ‚Äî the compiled component bundle + card index. The `*.card.html` previews reference `../../_ds_bundle.js`, so they won't render standalone locally without it.
- `templates/*/ds-base.js`, `templates/*/support.js` ‚Äî Claude Design `<x-dc>` runtime shims.
- per-component `*.d.ts` / `*.prompt.md` ‚Äî TypeScript types + design-agent usage docs.
- `templates/*/.thumbnail` ‚Äî preview images.
- `_adherence.oxlintrc.json` here omits the verbose generated `tokenKinds` map (kept the token list + component prop rules).

## Round-trip
- **Pull (refresh from Claude Design):** `DesignSync get_file` per path (no bulk export).
- **Push (publish edits):** `DesignSync finalize_plan` + `write_files` against the projectId.
  Tokens / CSS / `.dc.html` templates are plain source and take effect on push. **Component
  logic changes (`.jsx`) require the `_ds_bundle.js` rebuilt** ‚Äî set up the design-sync converter
  build first if you want true component round-trip (not done in this bootstrap).
